Swift - 字典中的增删改,遍历与合并

2016/11/12

增删改

var dict:[String:Any] = ["name" : "小明", "age" : 18, "sex":"man"]

// 新增
dict["title"] = "老大"
print(dict)

// 修改
dict["name"] = "赵四"
print(dict)

// 删除
dict.removeValue(forKey: "age")
print(dict)

遍历,合并

// 遍历
for e in dict {
    print(e)
}

for (k,v) in dict {
    print("key:\(k)  value:\(v)")
}
// 合并
var dict1:[String:Any] = ["name" : "李明", "age" : 18, "sex":"m"]
var dict2:[String:Any] = ["name" : "韩梅梅", "age" : 16, "sex":"w"]
// 如果把 dict2 合并到 dict1
for e in dict2 {
    dict1[e.key] = e.value   //dict2[e.key]
}
print("----------\n\(dict1)")

Swift - 数组中的增删改与合并

2016/11/12

第一篇写了数组遍历的几种情况,这篇主要就是数组中的增删改,也是开发中常用的

OC中数组分为可变(NSMutableArray)和不可变(NSArray) 相应的在Swift中就是 var & let

##追加元素
首先定义一个可变数组

var array = ["张三","李四","王五"]

追加一个元素

// 追加元素
array.append("赵六")
print(array)

##修改元素
通过下标进行修改

array[0] = "赵四"
print(array)

结果:

["赵四", "李四", "王五", "赵六"]

##删除

// 删除元素
array.remove(at: 0)
print(array)

结果:

["赵四", "李四", "王五", "赵六"]
["李四", "王五", "赵六"]

remove 的方法由很多,移除首/尾,移除范围...

array.removeAll(keepingCapacity: true)
print("数组中的元素\(array) 数组容量\(array.capacity)")

结果:

数组中的元素[] 数组容量6

##数组合并

var arr0 = [1,2,3]
let arr1 = [3,4,5]
arr0 += arr1
print(arr0)

结果:

[1, 2, 3, 3, 4, 5]

注意: 要合并的数组的类型必须一致,如果不一致会出错. 比如:

var arr0:[Any] = [1,2,3]
let arr1:[Any] = ["2",3,4,5]
arr0 += arr1
print(arr0)

Swift - 数组遍历

2016/11/12

最近没怎么更新,主要是最近刚入职新公司.正在熟悉环境中... 好了题目是Swift - 数组遍历 主要是项目要使用 Swift 开发,所以需要熟悉下语法.

##传统的下标遍历方法

let array = ["张三","李四","王五"]

// 第一种 下标遍历
for i in 0..<array.count {
    print(array[i])
}

##for in 遍历元素

// 第二种 for in 遍历元素
for s in array {
    print(s)
}

##enum block遍历

// 第三种 enum block 遍历
for e in array.enumerated() {
    print(e)
}

结果:

(0, "张三")
(1, "李四")
(2, "王五")

我们可以看下e的类型--元组

##元组方法遍历下标和内容
由上面第三种方法我们可以知道 e 的类型是元组,那么同样我们可以直接使用元组

// 第四种 元组使用
for (n,s) in array.enumerated() {
    print("\(n) \(s)")
}

结果:

0 张三
1 李四
2 王五

##反序遍历

// 第五种 反序遍历
for i in array.reversed() {
    print(i)
}

结果

王五
李四
张三

##反序遍历 带索引

// 第六种 反序遍历 带索引
for (n,s) in array.enumerated().reversed() {
    print("\(n) \(s)")
}

结果:

2 王五
1 李四
0 张三

注意: 先枚举再反序

总结:Swift 太灵活了...

##Scrum 快速迭代与反馈
快速迭代得到用户的反馈
##scrum概念
敏捷宣言:个体与交互 高于 流程与工具
工作的软件 高于 详尽的文档
客户合作 高于 合同谈判
相应变化 高于 遵循计划

在短时间内交付组稿价值的

Scrum框架 3 3 5 5
###3个角色
####PO(Product owner)产品经理

  • 一个人
  • 产品决策
  • 驱动产品走向成功
  • 客户与团队的桥梁(双向的代表)
  • 建立意愿
  • 负责"做什么"
  • 对Product Backlog 排优先级 (故事来表达条目,如果有其他的也可以)
  • 决定发布日期
  • 接受和退回工作成果

####开发团队 --> 规模 = 7+-2
能力:职能全(编码/测试),团队做一个项目
职责:产品增量(完整的可以上线的产品)
持续自我改进:所有成员都要有

####SM(Sturm Master)

  • 管理层与团队的桥梁(双向代表)
  • 帮助团队做决定,自己不做决定
  • 团队内部教练
  • 开放并善于聆听
  • 营造和推进Scrum价值观
  • 帮助团队移除障碍
  • 为PO和团队服务
  • 培养和保护团队 保护团队不受打扰
  • 提升管队效率

###3个工件(过程成果和交付物)

Product Backlog

  • 动态列表,包含产品功能(用户故事:)
  • 其他能给用户带来的价值的工作(技术故事:一般由主设计由产品经理确认)
  • 好的Product Backlog 详细,规模估算,涌现式,优先级(绝对序号排序,不要用高中低)
  • PO维护

####Sprint backlog

  • 基于product backlog 拆分成更小的任务.关注与怎么做的问题

  • 开发团队负责创建与维护

    1 选取要进行的Product backlog Item
    2 协作完成
    3 通过scrum任务版让团队自我管理Sprint Backlog
    4 通过燃尽图跟踪sprint进展情况并预测和管理开发的进展
    ####Sprint产品增量

  • 项目有一些列 sprint 组成

  • 通常2-4周

  • 固定周期 --> 让开发团队适应节奏

  • 时间盒结束sprint结束

  • 计划会定义的工作要在sprint内全部完成

###5个活动
####Sprint计划会

  • 第一部分:定义目标,选择可以承诺完成的任务(例如:主流程能跑通)
  • 第二部分:决定如何实现目标,创建Sprint backlog估算(故事清单,按清单做)
    ####Daily Scrum 每日站立会
  • 每日
  • 相同时间地点
  • 15分钟
  • 站立 时间端,累 注意力集中
  • 团队,+SM,PO

####迭代评审会

  • 展示工作成果,获得客户反馈
  • PO接受或者回退工作
  • 团队成员全部参家
  • 邀请相关干系人

####迭代回顾会

  • 团队开发过程会持续改进
  • 最长3小时
  • 全员参加,领导可以不参加
  • 敏捷教练或者SM带领大家回顾
  • 形成明确的改进计划,并在下一个迭代会回顾会检视效果
