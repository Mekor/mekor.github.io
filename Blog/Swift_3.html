iOS依赖库管理工具：Carthage

2016/11/14

新项目需要使用一种新的依赖库管理工具:Carthage

参见:iOS依赖库管理工具：Carthage

安装Carthage

  • 使用终端安装
$brew update
$brew install carthage
  • 在https://github.com/Carthage/Carthage/releases 下载最新版的Carthage.pkg

安装好Carthage后，查看Carthage当前版本

$brew carthage version

使用Carthage

  • 在项目所在文件夹中创建一个空的Carthage文件
$touch Cartfile  
$vi Cartfile

$touch Cartfile：新建一个名为Cartfile的文件

$vi Cartfile：打开名为Cartfile文件，如若该文件不存在会新建

  • 在Cartfile中加入一下内容，以Alamofire为例：
github "Alamofire/Alamofire" ~> 3.0

版本含义：
~> 3.0：表示使用版本高于3.0低于4.0的最新版本
== 3.0：使用3.0版本
>= 3.0：使用3.0版本或者更高版本
如果没有指明版本号，默认使用最新版本

  • 保存并关闭Cartfile文件，在终端执行如下命令，carthage开始下载和编译所需要的第三方库，当命令执行完毕，项目文件夹中会创建一个名为Carthage的文件夹，并且在~/Carthage/Build/iOS中出现Alamofire.framework文件
$carthage update

  • 打开项目，点击project->target->General，在Embedded Binaries添加Alamofire.framework

  • 此时已经可以使用导入的第三方库了，在需要的文件中导入头文件

Swift - 懒加载 lazy

2016/11/14

为什么使用懒加载?

  1. 能够延迟创建
  2. 解除解包的烦恼

参考Swifter中`lazy 修饰符和 lazy 方法

##常用方法

OC中的懒加载

// ClassA.h
@property (nonatomic, copy) NSString *testString;

// ClassA.m
- (NSString *)testString {
     if (!_testString) {
         _testString = @"Hello";
        NSLog(@"只在首次访问输出");
     }
     return _testString;
}

Swift中:

class ClassA {
    lazy var str: String = {
        let str = "Hello"
        print("只在首次访问输出")
        return str
    }()
}

简化版本:

lazy var str: String = "Hello"

##其他用法

        let data = 1...3
        let result = data.map { (i:Int) -> Int in
            print("正在处理\(i)")
            return 1*2
        }
        
        print("准备访问结果")
        
        for i in result {
            print("操作后结果为\(i)")
        }

结果:

正在处理1
正在处理2
正在处理3
准备访问结果
操作后结果为2
操作后结果为4
操作后结果为6

如果使用lazy

        let data = 1...3
        let result = data.lazy.map { (i:Int) -> Int in
            print("正在处理\(i)")
            return i*2
        }
        
        print("准备访问结果")
        
        for i in result {
            print("操作后结果为\(i)")
        }

结果:

准备访问结果
正在处理1
操作后结果为2
正在处理2
操作后结果为4
正在处理3
操作后结果为6

Swift - convenience 便利构造函数

2016/11/13

具体解释可以参见 Swifter 中Designated，Convenience 和 Required

使用遍历工造函数的好处可以参见如下代码:

import UIKit

extension UITextField {
    convenience init(frame:CGRect, placeholder:String,fontSize:CGFloat = 14) {
        self.init(frame:frame)
        
        self.borderStyle = .roundedRect
        self.placeholder = placeholder
        self.font = UIFont.systemFont(ofSize: fontSize)
    }
}

好处:

  1. 判断条件,只有满足条件才实例化对象,可以防止造成不必要的内存开销
  2. 简化对象的创建 ->利用extension抽取控件创建方法
  3. 本身不负责属性的创建和初始化工作
  4. 可以返回nil

注意点:

  1. 必须调用本类的 self.init,不能调用父类的super.init
  2. 不能被重写

Swift - KVC 和 运行时获取属性列表

2016/11/13

##Swift中如何使用KVC

import UIKit

class Person: NSObject {
    var name:String?
    var age: Int = 0
    var title:String?
    
    init(dict:[String:Any]) {
        super.init()
        setValuesForKeys(dict)
    }
    
    
    override func setValue(_ value: Any?, forUndefinedKey key: String) {
        
    }
}

注意:基本数据类型一开始要设置默认值.

##Swift中运行时获取属性列表

class Person: NSObject {
    var name:String?
    var age: Int = 0
    var title:String?
    
    init(dict:[String:Any]) {
        super.init()
        setValuesForKeys(dict)
    }
    
    
    override func setValue(_ value: Any?, forUndefinedKey key: String) {
        
    }
    
    class func propertyList() -> [String] {
        var count:UInt32 = 0
        
        let list = class_copyPropertyList(self,&count)
        
        var array:[String] = [];
        
        for i in 0..<Int(count) {
            
            // 获取属性
            let pty = list?[i]
            
            // 获取属性名称
            let cName = property_getName(pty)
            let name = String(utf8String: cName!)
            array.append(name!)
        }
        free(list)
        return array
    }
}

上面是OC写法,可以改进的更Swift一点:

    class func propertyList() -> [String] {
        var count:UInt32 = 0
        
        let list = class_copyPropertyList(self,&count)
        
        var array:[String] = [];
        
        for i in 0..<Int(count) {
            
            guard let pty = list?[i],
                let cName = property_getName(pty),
                let name = String(utf8String: cName)
                else {
                    // 继续遍历下一个
                    continue
            }
            array.append(name)
        }
        free(list)
        return array
    }

通过运行时可以发现:

  1. 基本数据类型如果设置成可选类型则运行时获取不到(在OC中没有可选).... 这就说明了上面的注意点,否则使用KVC会崩溃
  2. 如果属性为private同样获取不到属性(可以获取到ivar),同样会让KVC崩溃

Swift - 函数

2016/11/13

##函数的几种情况

// 正常
func sum(x:Int, y:Int) -> Int {
    return x+y
}
print(sum(x: 10, y: 20))

// 去掉形参
func sum(_ x:Int,_ y:Int) -> Int {
    return x+y
}
print(sum(10, 20))

// 外部参数
func sum(num1 x:Int,num2 y:Int) -> Int {
    return x+y
}
print(sum(num1: 10, num2: 20))

// 函数带默认值
func sum(x:Int = 1, y:Int = 2) -> Int {
    return x+y
}
print(sum())

##闭包

闭包类似于OC中的block,但应用更广. 与OC对比, OC中block是特殊的函数,在Swift中函数是特殊的闭包

// 最简单的闭包
// 定义
let foo = {
    print("最简单的闭包")
}
// 调用
foo()

// 带参数无返回值
let foo = { (x:Int) -> () in
    print(x)
}
foo(100)

// 带参数有返回值
let foo = { (x:Int) -> Int in
    return x+1
}
foo(10)

block常用在异步执行,回调...Swift中异步跟OC稍微有些区别.
###模拟异步加载

    /// 加载数据
    func loadData(completion:@escaping (_ result: [String])->()) {
        
        DispatchQueue.global().async {
            print("耗时操作\(Thread.current)")
            
            DispatchQueue.main.async {
                print("主线程更新\(Thread.current)")
                let json = ["头条","八卦","出大事了"]
                completion(json)
            }
        }
    }

注:关于**@escaping**参见 Swifter 中@escaping

###尾随闭包

        // 如果函数的最后一个参数是闭包,函数参数可以提前结束,最后一个参数直接使用{}包装闭包的代码
        loadData { (JSON) in
            print(JSON)
        }
